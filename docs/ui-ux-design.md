# NurseStudy - UI/UX設計書

> 看護学生のための「毎日3分」学習支援アプリ

---

## A. 全体像

### A-1. 情報設計（IA）

```
NurseStudy
├── Home（ホーム）
│   ├── 今日の3問カード
│   ├── 実習チェックカード（実習日のみ表示）
│   ├── 1行ふりかえりカード
│   └── 今日の達成サマリー
│
├── Quiz（クイズ）
│   ├── 問題表示
│   ├── 選択肢 / ○×
│   ├── 正誤フィードバック
│   ├── 解説
│   └── 苦手登録
│
├── Review（復習）
│   ├── 苦手カードスタック
│   └── わかる/まだ スワイプ
│
└── Practicum（実習）
    ├── チェックリスト
    └── テンプレ文章集
        ├── SBAR
        ├── 報告
        ├── 注意された時
        └── 患者対応
```

### A-2. 主要ユーザーフロー

#### フロー1: 通常日（今日の3問 → 完了）

```
[アプリ起動]
    ↓ (自動)
[Home画面] ← 今日やることが上から順に表示
    ↓ タップ1
[今日の3問カード]
    ↓ タップ2
[Quiz開始]
    ↓ 3問回答（各1タップ）
[結果表示] ← 静かな達成感
    ↓ タップ3
[Home画面] ← 3問完了マーク
    ↓ タップ
[1行ふりかえり] ← 気分選択 + 任意1行
    ↓
[今日完了] ← 所要時間: 約2-3分
```

#### フロー2: 実習日

```
[アプリ起動]
    ↓
[Home画面] ← 実習チェックカードが表示される
    ↓
[朝] 持ち物チェック
    ↓
[実習中] テンプレ文章参照（SBAR等）
    ↓
[夕方] 振り返りチェック
    ↓
[1行ふりかえり]
```

#### フロー3: 復習フロー

```
[Home or Reviewタブ]
    ↓
[苦手カードスタック表示]
    ↓
[スワイプ操作]
  → 右: わかった（苦手から外す）
  ← 左: まだ（残す）
    ↓
[全カード完了 or 途中離脱OK]
```

## B. 画面ごとの詳細仕様

### B-1. Home（ホーム画面）

#### 目的
- 今日やるべきことを一目で把握
- 3タップ以内で3問を開始
- 達成状況を静かに伝える

#### 主要コンポーネント一覧

| 優先度 | コンポーネント | 説明 |
|:---:|---|---|
| P0 | 今日の3問カード | メインCTA、最も目立つ |
| P0 | 達成インジケーター | 今日の進捗を視覚化 |
| P1 | 実習チェックカード | 実習日のみ表示 |
| P1 | 1行ふりかえりカード | 1日の締めくくり |
| P2 | 挨拶テキスト | 時間帯で変化 |
| P2 | 苦手復習への導線 | 控えめに配置 |

#### レイアウト構成

```
┌─────────────────────────────────┐
│ [セーフエリア: 44px]            │
├─────────────────────────────────┤
│                                 │
│ おはよう                    ←16px │
│ 今日も3分だけ、やってみよう      │  挨拶エリア (48px)
│                                 │
├─────────────────────────────────┤
│                                 │
│ ┌─────────────────────────┐    │
│ │  📝 今日の3問            │    │  メインカード
│ │                         │    │  (120px)
│ │  あと3問 →              │    │
│ └─────────────────────────┘    │
│                            ↑16px│
├─────────────────────────────────┤
│ ┌─────────────────────────┐    │
│ │  ✓ 実習チェック          │    │  サブカード1
│ │  今日の持ち物を確認 →    │    │  (80px)
│ └─────────────────────────┘    │  ※実習日のみ
│                            ↑12px│
├─────────────────────────────────┤
│ ┌─────────────────────────┐    │
│ │  💭 1行ふりかえり        │    │  サブカード2
│ │  今日の気持ちを残そう →  │    │  (80px)
│ └─────────────────────────┘    │
│                                 │
├─────────────────────────────────┤
│                                 │
│ 今日の達成  ○ ○ ○           │  達成サマリー
│            3問 実習 ふりかえり  │  (60px)
│                                 │
├─────────────────────────────────┤
│ [下タブ: 56px + セーフエリア]   │
└─────────────────────────────────┘
```

#### 状態設計

| 状態 | 表示内容 |
|---|---|
| 初回 | オンボーディング完了後、「はじめての3問」カード |
| 通常（未着手） | 「今日の3問」カード活性化、達成○は空 |
| 通常（一部完了） | 完了項目に✓、残りを強調 |
| 全完了 | 全○に✓、「おつかれさま」メッセージ |
| 実習日 | 実習チェックカードが追加表示 |
| 読み込み中 | スケルトンUI（0.5秒以内なら非表示） |
| エラー | 「読み込めませんでした」+ リトライボタン |

#### マイクロコピー例

**挨拶（時間帯別）**
- 5-11時: 「おはよう」
- 11-17時: 「おつかれさま」
- 17-22時: 「今日もおつかれさま」
- 22-5時: 「夜遅くまでえらいね」

**今日の3問カード**
- 未着手: 「あと3問」
- 1問完了: 「あと2問」
- 完了: 「今日の3問、完了！」

**励まし（完了時）**
- 「今日も3分、えらい」
- 「ちゃんと続けてるね」
- 「無理せず、これでOK」

**注意書き**
- 未完了でも: 「明日また来てね」（責めない）
- ストリーク切れ: 表示しない（罪悪感禁止）

### B-2. Quiz（クイズ画面）

#### 目的
- 3問を最短で回答できる
- 正誤と解説を簡潔に伝える
- 苦手登録でReviewに送る

#### 主要コンポーネント一覧

| 優先度 | コンポーネント | 説明 |
|:---:|---|---|
| P0 | 問題文 | 読みやすいサイズ |
| P0 | 選択肢ボタン | タップ領域大きめ |
| P0 | 正誤フィードバック | ○/×アイコン + 色 |
| P1 | 解説テキスト | 折りたたみ可 |
| P1 | 苦手登録ボタン | 「あとで復習」 |
| P1 | 進捗インジケーター | 1/3, 2/3, 3/3 |
| P2 | 次へボタン | 自動進行も可 |

#### レイアウト構成

```
┌─────────────────────────────────┐
│ [セーフエリア]                  │
├─────────────────────────────────┤
│ ← 閉じる           1/3 ○○○   │ ヘッダー (48px)
├─────────────────────────────────┤
│                                 │
│                                 │
│  成人の正常な脈拍数は           │
│  1分間に何回か？                │  問題文エリア
│                                 │  (フレキシブル)
│                                 │
├─────────────────────────────────┤
│                                 │
│  ┌─────────────────────────┐   │
│  │  A. 40〜60回            │   │  選択肢
│  └─────────────────────────┘   │  各56px
│  ┌─────────────────────────┐   │  マージン12px
│  │  B. 60〜100回  ← 正解   │   │
│  └─────────────────────────┘   │
│  ┌─────────────────────────┐   │
│  │  C. 100〜120回          │   │
│  └─────────────────────────┘   │
│  ┌─────────────────────────┐   │
│  │  D. 120〜140回          │   │
│  └─────────────────────────┘   │
│                                 │
├─────────────────────────────────┤
│ [下タブ: 非表示 - 集中モード]   │
└─────────────────────────────────┘
```

**回答後（正解時）**
```
┌─────────────────────────────────┐
│ ← 閉じる           1/3 ●○○   │
├─────────────────────────────────┤
│                                 │
│         ○ 正解！               │  フィードバック
│                                 │  (80px)
├─────────────────────────────────┤
│                                 │
│  正常な脈拍は60〜100回/分です。  │  解説
│  頻脈は100回以上、              │  (フレキシブル)
│  徐脈は60回未満を指します。      │
│                                 │
├─────────────────────────────────┤
│                                 │
│  [あとで復習する]               │  苦手登録
│                                 │  (48px)
├─────────────────────────────────┤
│                                 │
│  ┌─────────────────────────┐   │
│  │      次の問題へ          │   │  メインCTA
│  └─────────────────────────┘   │  (56px)
│                                 │
└─────────────────────────────────┘
```

#### 状態設計

| 状態 | 表示内容 |
|---|---|
| 問題表示 | 問題文 + 選択肢（タップ可能） |
| 回答中 | 選択した選択肢がハイライト |
| 正解 | ○アイコン + 正解色 + 解説表示 |
| 不正解 | ×アイコン + 不正解色 + 正解表示 + 解説 |
| 最終問題完了 | 結果サマリー画面へ遷移 |
| 読み込み中 | 問題文スケルトン |
| エラー | リトライボタン |

#### マイクロコピー例

**フィードバック**
- 正解: 「○ 正解！」
- 不正解: 「× おしい！」（責めない）

**苦手登録ボタン**
- 「あとで復習する」
- 登録後: 「復習リストに追加したよ」（Toast）

**完了時**
- 3/3正解: 「パーフェクト！」
- 2/3正解: 「いい調子！」
- 1/3正解: 「復習すれば大丈夫」
- 0/3正解: 「今日やっただけでえらい」

### B-3. Review（復習画面）

#### 目的
- 苦手問題だけを効率的に復習
- スワイプで直感的に仕分け
- 短時間で完了できる

#### 主要コンポーネント一覧

| 優先度 | コンポーネント | 説明 |
|:---:|---|---|
| P0 | カードスタック | スワイプ可能なカード |
| P0 | わかる/まだ ボタン | スワイプ代替 |
| P1 | 残りカード数 | 「あと5枚」 |
| P1 | 解説表示トグル | カードタップで開閉 |
| P2 | シャッフルボタン | 順番をランダムに |

#### レイアウト構成

```
┌─────────────────────────────────┐
│ [セーフエリア]                  │
├─────────────────────────────────┤
│ 復習                 あと 5 枚  │ ヘッダー (48px)
├─────────────────────────────────┤
│                                 │
│                                 │
│    ┌───────────────────────┐   │
│    │                       │   │
│    │  成人の正常な脈拍数は  │   │
│    │  1分間に何回か？      │   │  カード
│    │                       │   │  (280px)
│    │  ───────────────────  │   │
│    │                       │   │
│    │  答え: 60〜100回      │   │
│    │                       │   │
│    └───────────────────────┘   │
│         ↑ カードは前後に見える  │
│                                 │
├─────────────────────────────────┤
│                                 │
```

#### 状態設計

| 状態 | 表示内容 |
|---|---|
| 苦手あり | カードスタック表示 |
| 苦手なし | 「復習する問題はないよ」+ イラスト |
| 全カード完了 | 「おつかれさま！」+ Homeへ戻る |
| カード表（問題面） | 問題文のみ |
| カード裏（解答面） | タップで答え表示 |

#### マイクロコピー例

**空状態**
- 「復習する問題はないよ」
- 「クイズで『あとで復習』を押すとここに追加されるよ」

**完了時**
- 「今日の復習おわり！」
- 「スキマ時間にえらい」

**ボタン**
- 左: 「まだ」
- 右: 「わかった」

### B-4. Practicum（実習画面）

#### 目的
- 実習の準備・振り返りをサポート
- よく使うフレーズをすぐ参照
- チェックリストで抜け漏れ防止

#### 主要コンポーネント一覧

| 優先度 | コンポーネント | 説明 |
|:---:|---|---|
| P0 | チェックリスト | タップでチェック |
| P0 | テンプレ文章タブ | SBAR/報告/etc |
| P1 | 文章コピーボタン | 1タップでコピー |
| P1 | カテゴリ切り替え | タブまたはセグメント |
| P2 | カスタム項目追加 | チェックリスト編集 |

#### レイアウト構成

```
┌─────────────────────────────────┐
│ [セーフエリア]                  │
├─────────────────────────────────┤
│ 実習サポート                    │ ヘッダー (48px)
├─────────────────────────────────┤
│ [チェックリスト] [テンプレ文章] │ セグメント (44px)
├─────────────────────────────────┤
│                                 │
│  今日の持ち物                   │
│  ┌─────────────────────────┐   │
│  │ ☑ 実習着               │   │
│  │ ☑ 名札                 │   │
│  │ ☐ 聴診器               │   │  チェックリスト
│  │ ☐ ペンライト           │   │  (各48px)
│  │ ☐ メモ帳               │   │
│  │ ☐ 実習記録             │   │
│  └─────────────────────────┘   │
│                                 │
│  ＋ 項目を追加                  │
│                                 │
├─────────────────────────────────┤
│ [下タブ]                        │
└─────────────────────────────────┘
```

**テンプレ文章タブ**
```

│                                 │

│  ───────────────────────────── │
│                                 │
│  S: 状況（Situation）           │

│   現在〜の状態です」            │
│                                 │
│  B: 背景（Background）          │
│  「〇〇の既往があり、           │
│   昨日から〜」                  │
│                        [コピー] │
│                                 │
│  ─────────────────────────────  │
│                                 │
│  A: 評価（Assessment）          │
│  「〜と考えられます」           │
│                        [コピー] │
│                                 │
├─────────────────────────────────┤
│ [下タブ]                        │
└─────────────────────────────────┘
```

#### 状態設計

| 状態 | 表示内容 |
|---|---|
| 実習日ON | Homeにカード表示、Practicumタブ活性 |
| 実習日OFF | Practicumタブはアクセス可能だが控えめ |
| チェック完了 | 全項目✓で「準備OK！」 |
| テンプレ表示 | カテゴリごとにアコーディオン |
| コピー完了 | Toast「コピーしました」 |

#### マイクロコピー例

**チェックリスト**
- 全完了: 「準備OK！いってらっしゃい」
- 一部未完: 項目名を表示（責めない）

**テンプレ文章**
- コピーボタン: 「コピー」
- コピー後: 「コピーしました」（Toast, 1.5秒）

**注意された時（テンプレ）**
- 「すみません、確認不足でした」
- 「次からは〜するようにします」
- 「ご指導ありがとうございます」

| Variant | 用途 | スタイル |

|------|------|------------|----------|
| sm | 32px | 12px 16px | 14px |
| md | 44px | 12px 20px | 16px |
| lg | 56px | 16px 24px | 18px |

**状態**
- hover: 明度を5%下げる
- active: 明度を10%下げる + scale(0.98)
- disabled: opacity-50, cursor-not-allowed
- loading: spinner表示 + テキスト非表示

### D-3. 使い分けルール

#### ボタンの使い分け

| シーン | Variant | Size | 例 |
|--------|---------|------|-----|
| 画面のメインアクション | primary | lg | 「始める」「次へ」 |
| 補助的なアクション | secondary | md | 「あとで復習する」 |
| キャンセル・戻る | ghost | sm | 「閉じる」「スキップ」 |
| 削除（確認後） | danger | md | 「削除する」 |

#### Disabled状態の適用

```
適用する場面:
- 必須入力が未完了
- 処理中（loading状態と併用）
- 選択肢として無効

適用しない場面:
- 「できなかった」を責める用途 ← 罪悪感禁止の原則
```

#### Toast vs BottomSheet

| 情報の種類 | UI |
|------------|-----|
| 完了通知（軽い） | Toast |
| 選択が必要 | BottomSheet |
| エラー（軽い） | Toast |
| エラー（対応必要） | BottomSheet or モーダル |

## E. アクセシビリティ

### E-1. 文字サイズ

```
最小フォントサイズ: 14px（キャプションのみ12px許容）
本文推奨: 16px以上
見出し: 18px以上

理由:
- モバイルでの可読性確保
- 疲れ目の状態（夜間学習）でも読める
- システムフォントサイズ設定を尊重（rem使用）
```

### E-2. コントラスト比

```
WCAG AA準拠（最低基準）:
- 通常テキスト: 4.5:1 以上
- 大きいテキスト(18px+): 3:1 以上
- UI要素: 3:1 以上

実装値:
- neutral-800 on white: 12.6:1 ✓
- neutral-600 on white: 5.7:1 ✓
- primary-500 on white: 4.8:1 ✓
- white on primary-500: 4.8:1 ✓
```

### E-3. タップ領域

```
最小タップ領域: 44px × 44px（Apple HIG準拠）
推奨: 48px × 48px

実装方法:
- ボタン: min-height: 44px, padding調整
- チェックボックス: 視覚サイズ24px、タップ領域48px
- リンク: padding追加でタップ領域確保
- アイコンボタン: 44px × 44px以上
```

### F-2. 通知設計（ストレスを増やさない）

#### 基本方針

```
❌ やらないこと
- 「今日まだやってないよ！」（責める）
- 「ストリーク切れそう！」（焦らせる）
- 1日複数回の通知
- 夜遅く/早朝の通知

✅ やること
- 1日1回、ユーザーが選んだ時間に
- やさしいトーンで「誘う」だけ
- 通知をOFFにしても罪悪感なし
```

#### 設定UI

```
通知設定
─────────────────────────────
リマインダー        [ON/OFF]

時間   [19:00 ▼]

─────────────────────────────
※いつでもOFFにできます。
  OFFにしてもデータは消えません。
```

#### 通知文例

```
タイトル: NurseStudy
本文（ランダム）:
- 「今日の3分、いつでもどうぞ ☕」
- 「3問だけ、やってみる？」
- 「忙しい日はスキップしてOK」
- 「スキマ時間に、ちょっとだけ」

※「やらなきゃ」ではなく「やってもいいよ」のトーン
```

#### 技術実装メモ

```
- Web Push API（PWA）
- 通知許可は初回起動時ではなく、設定画面で明示的に
- ローカル通知（オフライン対応）
- 曜日別設定は複雑なのでMVP+1では省略
```
